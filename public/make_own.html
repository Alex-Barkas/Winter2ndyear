how to make your own.

A guide to configuring this academic dashboard for your own use.

This repository works as a complete operating system for your semester. It handles course links, assignment tracking, grade calculation, and daily planning without the bloat of traditional LMS platforms. Since you're likely using an AI assistant (like Google Gemini) to manage your code, this guide is written to help you direct that AI effectively.

1. Why This Architecture?

You might ask: "Why can't I just click 'Save' like in Notion or Google Docs?"

The answer: This is a Static Site. It has no database, no login system, and no monthly fees. It lives entirely as text files on GitHub.

  • VS Notion: You own the data. It loads instantly. You can customize every pixel.
  • VS Canvas/LMS: It contains only what you care about, not what the admin cares about.

Because there is no "backend" to save your clicks, you must tell your AI agent to update the code to make changes permanent. This might seem daunting, but it actually gives you absolute control.

2. Phase 1: Setup & Prerequisites

You don't need to be a developer to use this. You just need access to the source code and an LLM to do the heavy lifting.

  • Source Code: Fork the GitHub repository to your own account.
  • AI Assistant: Use "Antigravity" (Google Gemini) or a similar agentic coding tool.
  • Course Info: Have your syllabus PDFs and OnQ/portal links ready.

3. Phase 2: Configuration

The entire site is powered by a single file: public/student-config.js. This file contains the list of courses, assignments, and grading schemes.

3.1 Understanding the Data

Currently, the file is populated with my Winter 2026 courses, including MTHE 281 and MTHE 212. If you are taking these courses, you can keep them. Otherwise, you will need to swap them out for your own.

3.2 Prompting Your AI

Instead of editing the code manually, copy-paste the following prompt to your AI assistant to update the course list. Fill in the brackets with your details.

"I am configuring my student-config.js file.

1. Please keep the entries for MTHE 281 and MTHE 212 as I am taking those courses.
2. Remove the other courses (ENPH 239, CMPE 212, ELEC 274).
3. Add a new course: [COURSE CODE] - [COURSE NAME]. Here is the link to the OnQ page: [LINK].
4. Add a midterm for this new course on [DATE] at [TIME]."

The AI will automatically structure the data correctly, ensuring the new courses appear on the homepage and the deadlines appear in the calendar.

4. Phase 3: Assets & Features

To get the most out of the system, make sure you are using all the built-in tools. Here is what you need to provide to make them work.

4.1 The Weekly Schedule

The "weekly schedule" link in the navigation bar points to a file at public/schedule/schedule.pdf. This is intended to be your visual timetable (exported from SOLUS or a calendar app).

  • Action: Save your own schedule as a PDF and verify it is named schedule.pdf, then place it in the `public/schedule/` folder.
  • AI Prompt: "I have uploaded my new schedule.pdf. Please confirm it is in the correct directory so the link works."

4.2 Grade Dashboard

The grades.html page calculates your current standing based on weights defined in student-config.js. Only the courses defined in the gradingSchemes object will appear here.

  • Action: Ask your AI to update the `gradingSchemes` object with the weight distribution from your course syllabus (e.g., "Midterm 20%, Final 50%").

4.3 Planner

The planner.html page is a session-based focus tool. It uses the courses defined in your config file to let you tag study sessions. No setup required other than setting up your courses in Phase 2.

5. Phase 4: Personalization

Make the site feel like yours.

6. Phase 5: Workflow & Persistence

This system uses a "Code as Database" philosophy. This means there are two ways to interact with it:

  • Local (Temporary): Clicking "Mark as Complete" or entering a grade in the UI. This saves to your browser's local storage. Great for quick updates.
  • Permanent (Synced): Updating the student-config.js file. This creates a permanent record that syncs across devices.

6.1 Efficiency & Batching

One-off requests are slow. Don't ask your AI to update one assignment at a time. Batch your requests.

  • Bad: "Mark Lab 1 as done." (wait) "Now mark Lab 2 as done."
  • Good: "Update my status: I've finished Lab 1 (85%), Lab 2 (90%), and the Weekly Quiz. Also, add a reminder for my Dentist appointment on Friday."

Treat your AI like a secretary you meet with once a day. Dump all your updates in one prompt, and let it handle the code.

6.2 Google Drive Strategy

Since we don't have a database to store files, use Google Drive as your file server.

  1. Create a folder for each course in your Drive.
  2. Upload your PDFs, slides, and homework there.
  3. Get the "Share Link" (Anyone with link can view).
  4. Tell your AI: "Link the 'MTHE 281' button on the homepage to this Drive folder: [LINK]"

This keeps your repo light and fast while giving you unlimited storage for your actual documents.

6.3 Why This Matters

Disclaimer: Any changes you make in the browser UI (checking off a box, entering a grade) are temporary and live only on that device. To make it permanent, you must instruct your AI to update the config file.

This workflow separates "scratchpad" work (UI) from "official record" work (Code). It forces you to be intentional about your data, preventing the digital clutter that accumulates in apps like Notion.

Once you've completed these steps, you'll have a fully functional, personalized academic operating system.
